
**Zapret Dyson**

> Zapret Dyson — это аппаратно-ускоренный графический интерфейс (GUI) и система оркестрации для локального обхода систем глубокого анализа трафика (DPI). 
> 
> Проект построен на базе консольного перехватчика пакетов winws.exe (через сетевой драйвер WinDivert). В отличие от классических VPN-сетей, утилита не маршрутизирует трафик через сторонние удаленные серверы. Инъекция, фрагментация и модификация сетевых пакетов происходят локально на уровне ядра ОС. Данный подход позволяет обходить провайдерские блокировки, сохраняя 100% пропускной способности канала и оригинальный сетевой пинг.

**Архитектура и UI/UX**

> Проект полностью исключает необходимость использования командных оболочек (CMD/PowerShell) при повседневной эксплуатации:
> 
> * **Стек технологий:** Разработано на C++ с использованием DirectX 11 и библиотеки ImGui.
> * **Визуализация:** Интегрированы аппаратные шейдеры свечения и методы линейной интерполяции (Lerp) для отрисовки переходов интерфейса.
> * **Системная интеграция:** Окно утилиты связано с Win32 Region API для нативного скругления границ на уровне операционной системы.
> * **Оптимизация:** Потребление ресурсов центрального процессора (CPU) в режиме простоя составляет 0%.

**Технический функционал**

**Адаптивная фильтрация (Smart Profiles)**
> В логику приложения заложено 7 преднастроенных векторов атаки на DPI. Конфигурация выбирается пользователем в зависимости от типа применяемых провайдером ограничений:
> 
> * **Profile 1 & 2 (Classic):** Базовые протоколы fakedsplit и hostfakesplit. Эффективны для большинства магистральных сетей (Ростелеком, Метросеть, Дом.ру, ТТК).
> * **Profile 3 (Aggressive):** Алгоритм MultiSplit. Направлен на преодоление жестких блокировок (Уфанет, Билайн, Мегафон).
> * **Profile 4 & 5 (Mobile):** Узконаправленные алгоритмы SeqOvl и BadSeq. Спроектированы для обхода DPI мобильных операторов связи (Tele2, Yota, МТС).
> * **Profile 6 & 7 (Failsafe):** Конфигурации, обеспечивающие максимальный уровень десинхронизации и подмены пакетов в критических сетевых условиях.

**VoIP & Game Mode**
> Реализовано выделенное правило маршрутизации для диапазона портов 1024-65535 (TCP/UDP), включая специфицированный порт 1400. 
> 
> * Предотвращает потерю пакетов (Packet Loss) в многопользовательских сессиях.
> * Обеспечивает стабильность TCP/UDP-подключений к голосовым серверам Discord.
> * Устраняет задержки маршрутизации WebRTC-трафика в звонках Telegram.

**Native Windows Service**
> Обеспечена бесшовная инсталляция утилиты в качестве системной службы. Процесс перехватчика внедряется в автозагрузку ОС, функционирует в фоновом режиме до авторизации пользователя и не требует повторной ручной инициализации. Механизм включает автоматический парсинг абсолютных путей к конфигурационным базам.

**High-Priority Engine & Zero-Delay**
> * Процесс winws.exe инициализируется с флагом HIGH_PRIORITY_CLASS, что делегирует ему приоритетное право на обработку трафика планировщиком задач ОС. 
> * При каждом запуске алгоритм инициирует принудительный сброс кэша DNS (ipconfig /flushdns) и активирует параметр TCP Timestamps на уровне системы для моментального применения измененных маршрутов.

**Интегрированная диагностика**
> Внутренний модуль телеметрии осуществляет захват состояния перехватчика с выводом логов в графический интерфейс. Дополнительно поставляется модифицированный скрипт PowerShell (Diagnostic Toolkit) для стресс-тестирования сети и анализа доступности TCP-портов.

**Принципы продукта**

> 1. **Производительность:** Нулевое влияние на базовую задержку сети.
> 2. **Изолированность:** Полное отсутствие удаленных серверов. Модификация заголовков (SNI, TLS) производится исключительно на локальном хосте.
> 3. **One-Click Deployment:** Трансформация сложных конфигурационных параметров WinDivert в интерактивные элементы управления.
