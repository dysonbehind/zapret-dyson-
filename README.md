Zapret Dyson
Zapret Dyson (v1.4.8) — аппаратное решение с графическим интерфейсом для обхода систем глубокого анализа трафика (DPI).

Проект построен на базе консольного перехватчика пакетов winws.exe (через драйвер WinDivert). В отличие от VPN-сетей, утилита не маршрутизирует трафик через сторонние узлы. Инъекция, фрагментация и модификация сетевых пакетов происходят локально на уровне ядра ОС, что позволяет обходить блокировки провайдеров с сохранением 100% пропускной способности канала и оригинального пинга.

Архитектура и UX/UI
Проект полностью отходит от использования классических консольных окон (CMD/PowerShell) при повседневной эксплуатации.

Разработан графический интерфейс с аппаратным ускорением на базе DirectX 11 и библиотеки ImGui.

Внедрены интерполяционные анимации (Lerp) и шейдеры программного свечения.

Окно утилиты интегрировано с Win32 Region API для скругления системных границ.

Нагрузка на центральный процессор в режиме простоя составляет 0%.

Технический функционал
Адаптивная фильтрация (Smart Profiles)
В ядро вшито 7 преднастроенных векторов атаки на DPI. Конфигурации подбираются пользователем исходя из типа провайдера:

Profile 1 & 2: Базовые протоколы fakedsplit и hostfakesplit. Эффективны для магистральных сетей (Ростелеком, Метросеть, Дом.ру, ТТК).

Profile 3: Агрессивный алгоритм MultiSplit. Предназначен для жестких блокировок (Уфанет, Билайн, Мегафон, МТС).

Profile 4 & 5: Узконаправленные алгоритмы SeqOvl и BadSeq. Разработаны для мобильных операторов (Tele2, Yota, MTS).

Profile 6 & 7: Failsafe-конфигурации для максимального уровня десинхронизации и подмены пакетов.

VoIP & Game Mode
Встроенный туннель для портов 1024-65535 (TCP/UDP), включая жестко заданный порт 1400. Данный режим предотвращает потери пакетов (Packet Loss) в многопользовательских играх, восстанавливает доступ к голосовым серверам Discord и стабилизирует WebRTC-звонки в Telegram.

Native Windows Service
Реализована установка утилиты в качестве системной службы. Процесс обхода интегрируется в автозагрузку ядра, работает в фоновом режиме до авторизации пользователя и не требует ручного перезапуска. Аргументы командной строки парсятся с абсолютными путями к спискам хостов.

High-Priority Engine & Zero-Delay
Запуск процесса winws.exe происходит с параметром HIGH_PRIORITY_CLASS. Это гарантирует первоочередную обработку трафика процессором. Дополнительно программа инициирует принудительный сброс кэша DNS (ipconfig /flushdns) и активирует TCP Timestamps при каждом старте для моментального построения незаблокированных маршрутов.

Интегрированная диагностика
Встроенный модуль телеметрии транслирует состояние перехватчика в интерфейс программы. В комплект также входит переписанный PowerShell-скрипт (Diagnostic Toolkit) для стресс-тестирования сети и анализа доступности TCP-портов.

Принципы продукта
Производительность: Нулевое влияние на сетевые задержки.

Безопасность: Отсутствие удаленных серверов. Модификация заголовков (SNI, TLS) выполняется исключительно на хосте пользователя.

Концепция One-Click Deployment: Сведение сложных конфигураций WinDivert к выбору чекбокса в графическом меню.
