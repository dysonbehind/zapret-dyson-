> Zapret Dyson — это аппаратно-ускоренный графический интерфейс (GUI) и система оркестрации для локального обхода систем глубокого анализа трафика (DPI). 
> 
> Проект построен на базе консольного перехватчика пакетов `winws.exe` (через сетевой драйвер WinDivert). В отличие от классических VPN-сетей, утилита не маршрутизирует трафик через сторонние удаленные серверы. Инъекция, фрагментация и модификация сетевых пакетов происходят локально на уровне ядра ОС. Данный подход позволяет обходить провайдерские блокировки, сохраняя 100% пропускной способности канала и оригинальный сетевой пинг.

> [!WARNING]
> ### АНТИВИРУСЫ И РЕАКЦИЯ СИСТЕМЫ
> Программа использует драйвер `WinDivert64.sys` для работы с сетью на уровне ядра ОС. Некоторые антивирусы могут ложно срабатывать на этот драйвер (детект `Not-a-virus:RiskTool.Multi.WinDivert`), так как он модифицирует трафик. Это абсолютно нормальное поведение. Добавьте папку с утилитой в исключения антивируса для стабильной работы.

## 🚀 Архитектура и UI/UX

Проект полностью исключает необходимость использования командных оболочек (CMD/PowerShell) при повседневной эксплуатации.

* **Стек технологий:** Разработано на чистом C++ с использованием DirectX 11 и библиотеки ImGui.
* **Визуализация:** Интегрированы аппаратные шейдеры свечения и методы линейной интерполяции (Lerp) для отрисовки плавных переходов интерфейса.
* **Smart UI Synchronization:** Интерфейс умеет «общаться» с ядром Windows. При запуске он сканирует активные службы и автоматически восстанавливает статусы и галочки, синхронизируясь с реальным состоянием ОС.
* **Оптимизация:** Потребление ресурсов центрального процессора (CPU) в режиме простоя составляет ровно **0%**.

## 🛠 Технический функционал

### 1. Адаптивная фильтрация (Smart Profiles)
В логику приложения заложено 7 преднастроенных векторов атаки на DPI. Конфигурация выбирается в один клик в зависимости от типа применяемых провайдером ограничений:
* **Profile 1 & 2 (Classic):** Базовые протоколы `fakedsplit` и `hostfakesplit`. Эффективны для большинства магистральных сетей (Ростелеком, Метросеть, Дом.ру, ТТК).
* **Profile 3 (Aggressive):** Алгоритм `MultiSplit`. Направлен на преодоление жестких блокировок (Уфанет, Билайн, Мегафон).
* **Profile 4 & 5 (Mobile):** Узконаправленные алгоритмы `SeqOvl` и `BadSeq`. Спроектированы для обхода DPI мобильных операторов связи (Tele2, Yota, МТС).
* **Profile 6 & 7 (Failsafe):** Максимальный уровень десинхронизации и подмены пакетов в критических сетевых условиях.

### 2. Нативная интеграция Telegram (C++ WebSocket Proxy)
В ядро программы интегрирован собственный SOCKS5-прокси, написанный на чистом C++ с использованием системной криптографии Windows (`BCrypt`):
* **AES-CTR Дешифрация (MTProto):** Прокси аппаратно расшифровывает 64-байтный ключ инициализации Telegram, извлекая точный ID дата-центра и безошибочно маршрутизируя трафик.
* **Мгновенный TCP-Fallback:** Если провайдер глушит WebSocket, движок за миллисекунды (без разрыва) переключает трафик на прямой TCP-мост (отправляя проблемный узел в кулдаун на 60 сек).
* **Обход DNS (SNI Spoofing):** Программа подключается к чистым IP-адресам Telegram, динамически подменяя заголовки `Host` для прохождения TLS-хендшейка в обход блокировок DNS.
* **One-Click Setup:** Кнопка «Apply proxy to Telegram» генерирует URI-запрос (`tg://socks?server=...`) и применяет настройки в Telegram Desktop в один клик.

### 3. Native Windows Service (Фоновый режим)
Обеспечена бесшовная инсталляция утилиты в качестве системной службы. 
Процессы перехватчика (`zapret`) и Telegram Прокси (`ZapretTgProxy`) внедряются в ядро ОС. Они стартуют вместе с Windows и работают абсолютно невидимо, даже если главное окно приложения полностью закрыто. 

### 4. VoIP & Game Mode
Реализовано выделенное правило маршрутизации для диапазона портов **1024-65535 (TCP/UDP)**:
* Предотвращает потерю пакетов (Packet Loss) в мультиплеерных играх.
* Обеспечивает стабильность подключений к голосовым серверам **Discord** (включая порт 1400).
* Устраняет задержки маршрутизации WebRTC-трафика.

### 5. High-Priority Engine & Zero-Delay
* Процесс перехватчика инициализируется с флагом `HIGH_PRIORITY_CLASS`, что делегирует ему приоритетное право на обработку трафика планировщиком задач ОС. 
* Внедрен алгоритм `ReadExactly` (защита от TCP-фрагментации), предотвращающий краши туннелей на сетевом оборудовании провайдера.
* При запуске алгоритм инициирует сброс кэша DNS (`ipconfig /flushdns`) и активирует параметр TCP Timestamps для моментального применения измененных маршрутов.

## 📌 Принципы продукта

> 1. **Производительность:** Нулевое влияние на базовую задержку сети.
> 2. **Изолированность:** Полное отсутствие удаленных серверов. Модификация заголовков (SNI, TLS) производится исключительно на локальном хосте.
> 3. **One-Click Deployment:** Трансформация сложных консольных параметров в интерактивные элементы управления.

## 🗒 Добавление адресов прочих ресурсов
Список адресов для обхода можно расширить, добавляя их в текстовые файлы в папке `lists/`:
- `list-general-user.txt` — для доменов (поддомены учитываются автоматически).
- `list-exclude-user.txt` — для исключения доменов из фильтрации.
- `ipset-exclude-user.txt` — для исключения конкретных IP-адресов.
